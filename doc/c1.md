# Capítulo 1 – Enquadramento e Apresentação do Projeto

## 1.1 Enquadramento

A manutenção automóvel envolve a gestão de múltiplas entidades, como clientes, veículos, serviços e respetivas intervenções. Para além dos processos base de manutenção, é frequente que clientes mantenham preferências sobre veículos específicos, exigindo a criação de relações estruturadas adicionais.

O projeto **Car4Me**, desenvolvido no âmbito da unidade curricular **Desenvolvimento Web I**, tem como objetivo disponibilizar uma **API RESTful** para gerir operações de manutenção automóvel, facilitando o registo, consulta e organização da informação relevante. A API segue boas práticas de engenharia, utiliza tecnologias modernas e inclui documentação formal.

## 1.2 Descrição Sumária do Sistema

A API **Car4Me** permite gerir:

- Clientes  
- Veículos  
- Categorias de veículos  
- Serviços de manutenção  
- Registos de manutenções  
- **Veículos favoritos associados a clientes (relação muitos-para-muitos)**

O sistema foi desenvolvido em **Node.js + Express**, utilizando **MySQL** como base de dados relacional.  
A documentação é providenciada em **OpenAPI 3.0**, testável via interface Swagger.

## 1.3 Objetivo Geral

Criar uma API modular e escalável, adequada para gerir dados de clientes, veículos, categorias, serviços, manutenções e favoritos, com suporte a consultas avançadas via **filtros HTTP (query parameters)**.

## 1.4 Objetivos Específicos

- Desenvolver a API em **Node.js/Express**, com arquitetura modular;
- Modelar um sistema relacional em **MySQL**, incluindo:
  - Relações 1-N (Clientes → Veículos, Veículos → Manutenções);
  - Relações N-N (Clientes ↔ Veículos Favoritos);
- Implementar endpoints CRUD para todas as entidades;
- Implementar **filtros HTTP** para pesquisa (ex.: marca, estado, categoria...);
- Criar documentação completa em **OpenAPI 3.0**;
- Preparar execução via **Docker**;
- Elaborar um relatório técnico-académico detalhado.

## 1.5 Relações Principais do Domínio

O modelo conceptual baseia-se em várias relações essenciais:

- **Cliente (1) — (N) Veículo**  
  Cada cliente pode ter vários veículos registados.

- **Veículo (1) — (N) Manutenção**  
  Um veículo pode ter múltiplas manutenções.

- **Serviço (1) — (N) Manutenção**  
  Um serviço pode estar associado a várias manutenções.

- **Cliente (N) — (N) Veículo (Favoritos)**  
  Clientes podem marcar diferentes veículos como favoritos,  
  e um veículo pode ser favorito para múltiplos clientes.
